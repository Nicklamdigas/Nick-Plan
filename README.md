## 📋 Quick Overview

This repository contains a deliberate, step-by-step plan for Nicholas (21, 3rd-year Software Engineering student at Zetech University) to:

1. **Stop relying on AI to write code** and start *understanding* it
2. **Master Full-Stack Web Development** (Python + Web Dev focus)
3. **Build a professional portfolio** of real projects
4. **Enter the job market confidently** with a clear niche

### The Core Philosophy

> You are not bad at coding. You have never actually *learned* coding — you've been *generating* it.

The fix is not to stop using AI. The fix is to **change your relationship with AI**:
- ❌ Old way: "AI, write this for me" → copy → submit
- ✅ New way: "I'll write this myself" → AI reviews it → you understand the feedback

## ⚡ The Anti-AI-Crutch Protocol

These rules are **non-negotiable**. Break them consciously and you sabotage your own plan:

1. **Write code first, ask AI second**  
   Always attempt the problem yourself for 20-30 minutes before using Copilot/ChatGPT.

2. **Never paste code you can't explain**  
   If you can't explain it line by line, delete it and rewrite it until you can.

3. **Use AI like a senior dev, not a vending machine**  
   Ask "Why does this work?" not "Give me the code."

4. **Read the error message first**  
   Spend 5 minutes with the error, Google it, try to fix it, *then* ask AI.

5. **Comment your code before AI reviews it**  
   Writing comments forces you to understand what you wrote.

6. **Rebuild before you refactor**  
   Rebuild your vibecoded school app from scratch — understanding every line. This is your first real project.

## ❓ FAQ

**Q: What if I fall behind?**  
A: Adjust the plan. Update `plan-backup/session-records/session-plan.md` with realistic timelines. The plan adapts; your commitment doesn't.

**Q: Should I use AI for coding?**  
A: Yes, but follow the Anti-AI-Crutch Protocol. AI is a reviewer, not a writer.

**Q: How do I know if I'm learning?**  
A: You can explain your code. You've built something yourself. You've debugged without copy-pasting.

**Q: What if Zetech courses interfere?**  
A: Notify, but don't derail. Integrate Zetech work into this plan, don't let it replace it.
